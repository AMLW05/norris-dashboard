Course-Specific Guidance

üìö CSC-113: Introduction to Programming (Java)

Primary Tool: GitHub Copilot (FREE with Student Pack)

Why: IDE-integrated, prevents over-reliance, helps you learn syntax without doing the thinking for you.

Recommended Workflow:

  1. You: Write pseudocode or comments describing what you want
  2. Copilot: Suggests Java implementation
  3. You: Read, understand, and refactor the suggestion
  4. You: Test and verify it works as expected
  5. If stuck: Ask ChatGPT (FREE) "Explain this Java concept to me"

Learning Focus:

  • Understand EVERY line of code Copilot suggests
  • Use AI to learn syntax, not to avoid learning
  • Build muscle memory by typing, not just accepting
  • Ask "why" when Copilot suggests something unfamiliar

‚ö†Ô∏è Academic Integrity Guidelines

Using AI tools is ENCOURAGED at FTCC, but you must use them ethically:

  • ‚úÖ DO: Use AI to understand concepts and learn
  • ‚úÖ DO: Use AI as a tutor and debugging assistant
  • ‚úÖ DO: Ask AI to explain code you don't understand
  • ‚úÖ DO: Use AI suggestions as starting points, then customize
  • ‚ùå DON'T: Copy-paste AI solutions without understanding them
  • ‚ùå DON'T: Submit AI-generated code as 100% your own work
  • ‚ùå DON'T: Use AI to bypass learning fundamental concepts
  • ‚ùå DON'T: Rely on AI for exams or individual assessments

Golden Rule: If you can't explain and defend every line of code you submit, you didn't learn it.

‚ú® Best Practices for Both Courses

  • Start with pseudocode: Plan your logic BEFORE asking AI
  • Read every line: Never accept AI suggestions blindly
  • Ask "why": When AI suggests something, ask it to explain
  • Test everything: AI can be wrong - always verify
  • Refactor: Improve AI suggestions to match your style
  • Document: Comment your code to prove you understand it
  • Practice without AI: Regularly code without assistance to build skills
