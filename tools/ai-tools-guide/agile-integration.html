How AI Tools Enhance Your Agile Workflow

In professional software development, teams use Agile methodologies (like SCRUM) to organize their work into sprints, plan iteratively, and deliver value continuously. As a student developer, you'll follow similar processes in CSC-113 and CSC-114.

The key insight: Different AI tools excel at different phases of the development lifecycle. This guide shows you exactly when to use each tool for maximum productivity and learning. ALL FREE

Important: AI tools are assistants, not replacements. You're still the developer making decisions, writing code, and learning the fundamentals. Use AI to accelerate your workflow, not to bypass understanding.

CSC-113 vs CSC-114: Workflow Differences

CSC-113: Intro to OOP (C++)

Focus: Learning fundamentals, understanding concepts, building foundation

Recommended Tool Priority:

  • 1. Claude - Deep concept explanations, learning design patterns
  • 2. ChatGPT - Quick syntax help, debugging simple errors
  • 3. Copilot - Use sparingly, only for boilerplate after you understand
Strategy:

Focus on understanding over speed. Use AI to explain concepts, not just write code. Try solving problems yourself first, then verify with Claude.

CSC-114: Advanced OOP (C++/Java)

Focus: Complex systems, design patterns, professional practices, speed

Recommended Tool Priority:

  • 1. Copilot - Fast implementation, you know what to build
  • 2. Claude - Architecture decisions, design pattern selection
  • 3. ChatGPT - Debugging, testing, quick problem-solving
Strategy:

Maximize productivity. Use Copilot heavily for implementation, Claude for architecture. You should understand the fundamentals by now, so focus on building complex systems efficiently.

Key Difference:

In CSC-113, AI is primarily a teaching assistant. In CSC-114, AI becomes a productivity multiplier. The shift happens because you've built foundational understanding in 113, so 114 is about applying those concepts at scale and speed.

Start with Planning

Never jump straight to coding. Use Claude to plan your approach first. 5 minutes of planning saves hours of refactoring.

Why Planning Matters:

  • Prevents architectural mistakes that are expensive to fix later
  • Helps identify design patterns early (Factory, Observer, Strategy)
  • Clarifies requirements before you write a single line

How to Do It:

  • Open Claude and describe your project/assignment
  • Ask: "What's the best class structure for this problem?"
  • Ask: "What design patterns would fit this scenario?"
  • Get a plan BEFORE opening your IDE

Example Prompt:

"I'm building a library management system for CSC-114. It needs to manage books, borrowers, due dates, and fines. What class structure and design patterns should I use? Please explain using SOLID principles."

Switch Tools Intentionally

Don't stick with one tool out of habit. Each tool has strengths - use the decision tree above to choose the right tool for each task.

When to Switch:

  • Start with Copilot? Switch to Claude if you're confused about architecture
  • Using ChatGPT? Switch to Claude if you need to analyze your entire codebase
  • Using Claude? Switch to Copilot once you know what to build

Decision Framework:

  • Planning phase: Claude (architecture, design patterns)
  • Coding phase: Copilot (real-time suggestions in IDE)
  • Stuck on concept: ChatGPT (explanations, tutorials)
  • Complex debugging: Claude (deep analysis)
  • Research needed: Gemini (documentation, best practices)

Pro Tip:

Use the Usage Tracker to log which tool you used for each task. After 10 tasks, review your patterns and see if you're choosing intentionally or just using what's easiest.

Understand Before Using

Never paste AI-generated code without understanding it. Ask the AI to explain any code you don't fully comprehend.

The Understanding Test:

  • Can you explain each line to someone else?
  • Do you know WHY the AI chose this approach?
  • Could you write similar code from scratch?
  • If you can't answer YES to all three, ask for explanation!

How to Learn from AI Code:

  • Step 1: Get the code from AI
  • Step 2: Ask "Explain each part of this code like I'm learning it for the first time"
  • Step 3: Ask "Why did you choose this approach over alternatives?"
  • Step 4: Modify the code yourself to test understanding

Example Follow-Up Prompt:

"Thanks for the code! Before I use it, can you explain: 1) What does line 15 do and why? 2) Why did you use a HashMap instead of an ArrayList? 3) What would break if I removed the null check?"

Review AI Suggestions

AI can make mistakes. Always review suggestions critically. Use Claude to review code from Copilot or ChatGPT.

Common AI Mistakes:

  • Outdated practices: AI may suggest old patterns
  • Over-engineering: Complex solutions for simple problems
  • Edge cases missed: Null checks, empty arrays, boundary conditions
  • Security issues: SQL injection, XSS vulnerabilities

Review Checklist:

  • ‚úÖ Does it compile/run?
  • ‚úÖ Does it handle edge cases (null, empty, negative)?
  • ‚úÖ Is it following course requirements (e.g., "must use Factory pattern")?
  • ‚úÖ Is it readable and maintainable?
  • ‚úÖ Are there any security concerns?

Cross-Check Strategy:

Best Practice: Got code from Copilot? Paste it into Claude and ask: "Review this code for bugs, edge cases, and best practices. What would you improve?"

Copilot for Speed, Claude for Depth

Use Copilot when you know exactly what to build. Use Claude when you need to figure out what to build or why.

Copilot Strengths (Speed):

  • Real-time suggestions as you type
  • Great for boilerplate (getters, setters, constructors)
  • Autocompletes based on context of current file
  • Fast implementation once you know the approach

Claude Strengths (Depth):

  • Reads your entire codebase (200K tokens)
  • Suggests design patterns and architecture
  • Explains trade-offs between approaches
  • Deep analysis of complex bugs

Perfect Workflow:

  • Phase 1 (Planning): Ask Claude "How should I structure this?"
  • Phase 2 (Implementation): Use Copilot in your IDE to write the code
  • Phase 3 (Review): Paste back to Claude "Does this follow best practices?"

When to Switch Mid-Task:

In Copilot: Getting suggestions but confused? ‚Üí Switch to ChatGPT/Claude
In Claude: Got a clear plan? ‚Üí Switch to Copilot for fast coding

Learn from AI

Post-Task Learning Questions:

  • "Explain the algorithm you just used in simple terms"
  • "What's the time complexity of this solution and why?"
  • "What are the pros and cons of this approach?"
  • "When should I use this pattern vs. alternatives?"

Build Exam Readiness:

  • After AI helps solve a problem, ask for 3 similar practice problems
  • Request conceptual explanations you can study
  • Ask for common mistakes to avoid

Example Learning Prompt:

Test Everything

AI-generated code needs testing just like human code. Use ChatGPT to generate comprehensive test cases.

Testing Workflow:

  • Step 1: Get code from AI (Copilot, ChatGPT, Claude)
  • Step 2: Ask ChatGPT: "Generate JUnit tests for this code"
  • Step 3: Run the tests and verify they pass
  • Step 4: Add edge case tests the AI missed

Types of Tests to Request:

  • Normal cases: Expected inputs and outputs
  • Edge cases: Empty, null, zero, negative, boundary values
  • Error cases: Invalid inputs that should throw exceptions

Example Test Request:

"Generate JUnit test cases for this Calculator class. Include tests for: 1) Normal operations, 2) Division by zero, 3) Null inputs, 4) Integer overflow. Explain what each test validates."

Collaborate with AI

Treat AI as a junior developer. Give it clear instructions, review its work, and provide feedback to get better results.

Effective AI Collaboration:

  • Clear Instructions: Specify language, requirements, constraints
  • Iterative Refinement: "Good start, but change X to Y"
  • Code Reviews: Point out issues and ask for fixes
  • Learning Mode: Ask AI to explain its decisions

Junior Developer Mindset:

  • Would you accept code from a junior without review? No ‚Üí Same with AI
  • Would you blindly trust a junior's architecture? No ‚Üí Same with AI
  • Would you expect perfect code on first try? No ‚Üí Iterate with AI

Feedback Loop Example:

First: "Write a Java class for managing inventory"
Review: "Good, but add input validation and use HashMap instead of ArrayList"
Refine: "Now add exception handling for negative quantities"
Final: Verify and customize for your needs

Know When NOT to Use AI

Simple problems you can solve in 2 minutes? Do it yourself. Build your skills first, use AI for acceleration, not replacement.

When to Code Yourself:

  • Simple syntax: Basic for loops, if statements, variable declarations
  • Learning phase: First 3 times doing something new (build muscle memory)
  • Exams/quizzes: Academic integrity requires your own work
  • Quick 2-minute tasks: Asking AI takes longer than just doing it

Skill Building First:

  • Week 1-3 CSC-113: Minimal AI (learn basics manually)
  • Week 4+: Start using AI for speed, but understand everything
  • Before midterm: Practice coding without AI to verify you've learned

The 10-Minute Rule:

Try it yourself for 10 minutes first. If stuck, THEN use AI. This ensures you're learning, not just getting answers. You'll be surprised how much you can figure out!

Iterate on Prompts

First AI response not quite right? Refine your prompt. Good prompting is a skill - practice makes perfect.

Iteration Strategies:

  • Start broad: "Explain inheritance in Java"
  • Add specificity: "Focus on the difference between extends and implements"
  • Request examples: "Show me with Animal/Dog/Cat classes"
  • Adjust complexity: "Explain like I'm a beginner" or "Include advanced topics"

Common Refinements:

  • "Make it simpler" / "Add more detail"
  • "Show code examples" / "Just explain conceptually"
  • "Use CSC-113 concepts only" / "Include design patterns"
  • "What's the trade-off?" / "Why did you choose this approach?"

Prompt Iteration Example:

V1: "Help with arrays" (Too vague)
V2: "Explain how to sort an array in Java" (Better)
V3: "Show me bubble sort vs. Arrays.sort() with pros/cons" (Specific)
V4: "Now explain when I should use each in CSC-113 assignments" (Perfect)

Remember: You're the Developer

AI tools are incredible productivity multipliers, but they're assistants, not replacements. Your job is to:

  • Make architectural decisions
  • Understand the code you're writing
  • Verify AI suggestions are correct
  • Learn concepts deeply for exams
  • Think critically about design trade-offs

The best developers use AI to handle repetitive tasks so they can focus on creative problem-solving, architecture, and learning. That's exactly what this workflow helps you do.
