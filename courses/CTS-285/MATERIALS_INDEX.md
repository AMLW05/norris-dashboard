## Content Quality Standards

All materials follow CSC-113/CSC-114 standards:

### Assignments
- ‚úÖ 300-1400 lines depending on complexity
- ‚úÖ Andrew's authentic voice (encouraging, practical, honest)
- ‚úÖ Learning objectives, prerequisites, deliverables checklist
- ‚úÖ Common issues & solutions section
- ‚úÖ "Andrew's Note" with teaching philosophy
- ‚úÖ AlgoCratic framing in collapsible sections (optional engagement)
- ‚úÖ Clear grading rubrics (40% Process, 40% Technical, 20% Communication)

### Knowledge Checks
- ‚úÖ 12-15 multiple choice questions per module
- ‚úÖ 60% SDLC concepts, 40% PM practices
- ‚úÖ Scenario-based, application-focused
- ‚úÖ Explanations that teach, not just answer
- ‚úÖ Unlimited attempts, auto-gradable
- ‚úÖ 15-20 minute completion time

### Activities
- Expected: 500-900 lines each
- Expected: 90-180 minute hands-on labs
- Expected: Materials needed, learning objectives, timed parts
- Expected: Instructor notes with timing, common struggles, teaching tips
- Expected: Extension challenges for advanced students

### Planning Sheets
- Expected: Purpose, when to use, instructions, template, filled example
- Expected: "Why this matters" connection to professional practice
- Expected: Common mistakes to avoid
- Expected: Integration with course project

### Reference Materials
- Expected: Category organization, Ctrl+F friendly
- Expected: Plain language definitions with examples
- Expected: Code snippets copy-paste ready
- Expected: Troubleshooting with symptoms, cause, solution, prevention

- ‚úÖ SDLC phases covered across all 8 modules

## How to Use This Course

### For Instructors

1. **Start with README.md** - Overview of course structure and philosophy
2. **Review INSTRUCTOR_GUIDE.md** (when complete) - Teaching strategies and grading shortcuts
3. **Import Canvas HTML pages** - Module structure with embedded materials
4. **Customize assignments** - Adapt examples to your institution's context
5. **Set up GitHub Classroom** - Automate repository creation for students
6. **Emphasize optional Algocratic elements** - Not required for success

### For Students

1. **Week 1**: Complete environment setup immediately
2. **Week 2**: Choose semester project (portfolio-worthy!)
3. **Modules 1-2**: Foundation - SDLC and requirements
4. **Modules 3-5**: Three complete sprints (design, implement, refine)
5. **Modules 6-7**: Quality and deployment
6. **Module 8**: Showcase your work, prepare for CTS-289

### For Course Reviewers

- **CLO/MLO alignment**: See program-data.json (when complete)
