üî¨ ML Evaluation Metrics You'll Implement

  • Accuracy: Overall correctness (but beware imbalanced datasets!)
  • Precision: Of predicted positives, how many are actually positive?
  • Recall: Of actual positives, how many did we catch?
  • F1-Score: Harmonic mean of precision and recall
  • ROC Curve & AUC: Visualize true positive vs. false positive rate
  • Confusion Matrix: See all classification outcomes
  • Cross-Validation: k-fold validation for robust evaluation

üí° Drew's First Sprint Survival Guide

  • Don't over-commit: Sprint 1 is about learning the process. 15 story points is plenty.
  • Standups take 2 minutes: Keep them brief. "Yesterday I did X. Today I'll do Y. Blocked on Z (or nothing)."
  • Burndown doesn't have to be perfect: It's a learning tool. Update it honestly, even if it looks messy.
  • Start technical work Day 2: Don't spend all of Week 5 planning. Get coding early.
  • Retrospective is sacred: This is where real learning happens. Be honest, not performative.
  • ROC curves will confuse you at first: That's normal. Read the docs, try examples, ask questions.
  • Your velocity will be low: Sprint 1 velocity is always low because you're learning. That's expected!

‚ö†Ô∏è Common Sprint 1 Pitfalls

  • Ignoring PM deliverables until the last day: PM is 50% of your grade! Don't treat it as an afterthought.
  • Not actually using the sprint board: I can tell if you updated it all at once on Day 10. Use it daily!
  • Vague retrospectives: "Do better next time" isn't actionable. Be specific!
  • Over-estimating your capacity: If in doubt, plan fewer stories. Better to finish early than carry over.
  • Skipping standups: Need 8/10 for full credit. Set a phone reminder!
  • Confusing accuracy with goodness: 90% accuracy on imbalanced data can be terrible. Use multiple metrics!
