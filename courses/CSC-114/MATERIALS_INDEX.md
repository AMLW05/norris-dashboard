## Usage Notes for Instructors

### First-Time Preparation
1. Read Instructor Guide (`instructor-guide/INSTRUCTOR_GUIDE.md`)
2. Review all 16 assignments in sequence
3. Set up GitHub organization for student repos
4. Prepare sprint board examples (Trello, GitHub Projects, or Jira)
5. Review PM mini-lessons (integrate into lectures)

### Time Investment Estimates
- **Prep**: 2 hours/week (lecture + activity setup)
- **Grading**: 1-1.5 hours/student/sprint (faster with practice)
- **Office Hours**: Expect high traffic Weeks 5-8 (first sprints)

### Common Student Challenges by Week
1. **Weeks 1-2**: Environment setup issues (Python, Jupyter, Git)
2. **Weeks 3-4**: Understanding story points ("How many is this worth?")
3. **Weeks 5-6**: First sprint overwhelming (expect this!)
4. **Weeks 7-8**: Sprint fatigue (encourage through it!)
5. **Weeks 9+**: PM becomes automatic (smooth sailing)
