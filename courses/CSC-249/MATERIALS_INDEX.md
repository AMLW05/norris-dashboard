## Unique Course Features Inventory

- **Module 1**: Stack implementation BEFORE reading Stack theory

- **Module 3**: Sorting implementations BEFORE divide-and-conquer theory

- **AI Prompt Documentation Template**: Structured template for documenting AI interactions

- **AI Collaboration Reflection**: Capstone includes comprehensive AI learning process analysis
- **Ethics Guidelines**: Transparent AI use required, hiding collaboration prohibited

- **All assignments**: Starter code provided in BOTH C++ and Python

- **Velocity tracking**: Sprint-over-sprint velocity analysis required
- **Process Portfolio**: Module 8 includes comprehensive analysis of all 4-5 sprints

## Usage Notes for Instructors

### First-Time Preparation
1. Read COURSEMAP.md thoroughly (understand create-first pedagogy)
2. Review all 14 assignments in sequence (understand progression)
3. Set up GitHub organization for student repos
4. Prepare sprint board examples (GitHub Projects recommended)
5. Review AI collaboration guidelines (prompts.txt expectations)
6. Choose visualization tools to demonstrate (VisuAlgo, Python Tutor)
7. Prepare dual-language environment support (C++ and Python setup guides)

### Time Investment Estimates
- **Prep**: 2-3 hours/week (lecture + activity setup + dual-language support)
- **Grading**: 1.5-2 hours/student/sprint (technical + PM + AI collaboration review)
- **Office Hours**: Expect high traffic Weeks 7-10 (first two sprints)
- **AI Collaboration Oversight**: 30 min/week reviewing prompts.txt files

### Common Student Challenges by Week
1. **Weeks 1-2**: Environment setup (C++ compiler, Python, Git, IDE)
2. **Weeks 3-4**: Understanding Big O empirically vs theoretically
3. **Weeks 5-6**: Implementing merge sort and quick sort recursively
4. **Weeks 7-8**: First sprint overwhelming (pointer bugs in linked list, sprint planning)
5. **Weeks 9-10**: Hash collision resolution strategies
6. **Weeks 11-12**: Tree traversal recursion and pointer manipulation
7. **Weeks 13-14**: Graph algorithm complexity
8. **Weeks 15-16**: Capstone scope management
