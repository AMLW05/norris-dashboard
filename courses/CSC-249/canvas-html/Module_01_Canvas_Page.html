Drew's Tips for Success

  • Build first, understand later: Don't read about Stacks for hours. Build one, break it, fix it, then understand why it works.
  • Array-based vs node-based: Start with array-based Stack (simpler). Try node-based if you're comfortable with pointers.
  • AI prompting strategy: Start with "Explain how a Stack works" then "Help me implement push()" then "Why did my pop() crash?"
  • Commit after every function: push() works? Commit. pop() works? Commit. Test passes? Commit.
  • prompts.txt is evidence: Document every AI conversation. This proves you're learning, not copying blindly.
  • Undo systems are everywhere: Ctrl+Z, browser back button, Git itself—all use Stack. You're building real software patterns.

Drew's Note

Welcome to CSC-249! This course is different from CSC-134. We're not learning syntax anymore—we're building professional development muscles. The GitHub workflow feels bureaucratic at first ("Why do I need a PR for my own code?"), but trust the process. By Module 4, you'll understand why.

AI collaboration is mandatory, not cheating. Every assignment requires prompts.txt documentation. Use Claude, ChatGPT, or GitHub Copilot—I don't care which. What matters is that you can explain the code in your own words. If AI writes something you can't explain, you haven't learned it yet.

The Stack implementation in Week 1 is intentionally simple. We're building your data structure muscles gradually. Enjoy the satisfaction of "it compiles!" while it lasts—Module 3 sorting algorithms will humble you.
