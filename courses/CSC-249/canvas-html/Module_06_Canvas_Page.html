Code Preview: Binary Search Tree

Recursive tree operations:

struct Node {
    int data;
    Node* left;
    Node* right;
    Node(int val) : data(val), left(nullptr), right(nullptr) {}
};

class BST {
private:
    Node* root;

    Node* insertHelper(Node* node, int val) {
        if (node == nullptr) {
            return new Node(val);
        }
        if (val < node->data) {
            node->left = insertHelper(node->left, val);
        } else {
            node->right = insertHelper(node->right, val);
        }
        return node;
    }

    void inorderHelper(Node* node) {
        if (node == nullptr) return;
        inorderHelper(node->left);
        std::cout << node->data << " ";
        inorderHelper(node->right);
    }

public:
    BST() : root(nullptr) {}

    void insert(int val) {
        root = insertHelper(root, val);
    }

    void inorderTraversal() {
        inorderHelper(root);
        std::cout << std::endl;
    }
};

Key insight: BST maintains order property (left < parent < right). Inorder traversal visits nodes in sorted order. O(log n) average for balanced trees.

Drew's Tips for Success

  • Trees are recursion in disguise: Every tree operation uses recursion naturally. Master it here.
  • Inorder = sorted: Inorder traversal of BST gives sorted output. This is why BSTs matter.
  • Heap != BST: Heaps maintain parent-child relationship (parent >= children), not left < right. Different use cases.
  • Heaps enable priority queues: O(log n) insert/extract makes heaps perfect for task scheduling.
  • Expression trees are compiler fundamentals: Understanding them unlocks how programming languages work.
  • Sprint 3 autonomy: Less hand-holding this sprint. You should know the workflow by now. Ask for help if you don't.
  • Velocity should stabilize: Sprint 1 = learning. Sprint 2 = adjusting. Sprint 3 = consistent delivery.

Drew's Note

Trees are where data structures get beautiful. BSTs give you O(log n) search with sorted order. Heaps give you O(log n) priority access. Expression trees show you how compilers parse math. This module is dense—pace yourself.
