Project Example: Social Network Analyzer

Data Structures Used:

  • Graph (Adjacency List): Represent user connections, friendships
  • Hash Table: Store user profiles, O(1) lookup by username
  • Heap (Priority Queue): Find trending users (most connections) or content

Features Implemented:

  • Add users and connections (graph operations)
  • Find degrees of separation (BFS shortest path)
  • Suggest friends (mutual connections)
  • Find influencers (users with most connections using heap)
  • Detect communities (connected components)

Complexity Analysis:

  • Add user: O(1) using hash table
  • Add connection: O(1) using adjacency list
  • Find path: O(V + E) using BFS
  • Find top influencer: O(V log V) using heap

Drew's Tips for Capstone Success

  • Choose a project you care about: Passion shows in presentations. Pick something that excites you.
  • Start simple, add complexity: Build core functionality first, then add advanced features.
  • Justify every structure: Don't use BST "because it's cool." Use it because sorted order matters for your app.
  • Test extensively: Comprehensive tests demonstrate professionalism. Edge cases matter.
  • Portfolio evidence: Include screenshots of burndown charts, velocity graphs, retrospective insights.
  • Sprint 5 is autonomous: Plan it yourself using 4 sprints of data. This proves mastery.
  • Presentation practice: Rehearse your demo. Technical glitches happen—have backup plans.
  • AI collaboration transparency: Show how AI helped you learn, not replace learning. Explain code in your words.
