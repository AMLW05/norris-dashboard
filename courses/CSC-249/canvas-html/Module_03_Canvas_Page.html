Code Preview: Merge Sort (Divide & Conquer)

See how divide-and-conquer works:

void merge(int arr[], int left, int mid, int right) {
    int n1 = mid - left + 1;
    int n2 = right - mid;
    int L[n1], R[n2];

    // Copy data to temp arrays
    for (int i = 0; i < n1; i++) L[i] = arr[left + i];
    for (int j = 0; j < n2; j++) R[j] = arr[mid + 1 + j];

    // Merge temp arrays back
    int i = 0, j = 0, k = left;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) {
            arr[k] = L[i];
            i++;
        } else {
            arr[k] = R[j];
            j++;
        }
        k++;
    }

    // Copy remaining elements
    while (i < n1) { arr[k] = L[i]; i++; k++; }
    while (j < n2) { arr[k] = R[j]; j++; k++; }
}

void mergeSort(int arr[], int left, int right) {
    if (left < right) {
        int mid = left + (right - left) / 2;
        mergeSort(arr, left, mid);      // Sort left half
        mergeSort(arr, mid + 1, right); // Sort right half
        merge(arr, left, mid, right);   // Merge sorted halves
    }
}

Key insight: Merge sort divides the array in half recursively, sorts each half, then merges. O(n log n) in all cases.

Drew's Tips for Success

  • Visualize before coding: Watch VisuAlgo animations first. Understanding the pattern makes implementation easier.
  • Selection sort is the simplest: Find minimum, swap, repeat. Build this one first to gain confidence.
  • Insertion sort is like sorting cards: Pick each element and insert it into the sorted portion. Natural for partially sorted data.
  • Merge sort never fails: O(n log n) in all cases, but uses extra memory. Great for linked lists.
  • Quick sort is fastest on average: O(n log n) average, but O(n²) worst case (if pivot selection is bad). Industry standard.
  • Benchmark reveals truth: Theory says O(n log n) beats O(n²). Measure it yourself and see the difference at 50,000 elements.
  • Story points are relative: If selection sort is 3 points, merge sort might be 8 (more complex, recursive, debugging time).

Drew's Note

Sorting algorithms are the "Hello World" of computer science interviews. You'll implement them here, analyze them, benchmark them, and understand why O(n log n) beats O(n²) at scale. This isn't theoretical—you'll graph the performance difference at 50,000 elements and see quick sort destroy selection sort.

Divide-and-conquer is everywhere. Merge sort and quick sort use the same pattern: break the problem in half, solve each half, combine the results. You'll see this pattern again in trees, graphs, and advanced algorithms. Master it now.
